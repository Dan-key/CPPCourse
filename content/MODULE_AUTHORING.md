# Правила написания модуля курса «Трек системного программиста»

> Это спецификация для автора модуля (человека или ИИ). Соблюдай её дословно:
> фронтенд и прогонщик жёстко завязаны на описанные здесь форматы. Нарушение
> формата = «md поехала» или «квиз не грузится».

---

## 0. Принцип: ОБЪЁМА ДОЛЖНО БЫТЬ МНОГО

Курс рассчитан на ~1100 часов. Каждый модуль — это «по книжке», а не конспект.

**Жёсткие минимумы на один модуль:**

| Артефакт | Минимум | Цель |
|----------|---------|------|
| Лекция (`.md`), строк | **2000** | 2000–2800 |
| Упражнений (папок) | **4** | 4–6 |
| Квиз: уровень БАЗА (mcq) | **8** | 8–12 |
| Квиз: уровень МЕХАНИЗМЫ (self_grade) | **8** | 8–10 |
| Квиз: уровень ЭКСПЕРТ (self_grade) | **5** | 5–7 |
| Квиз: уровень ЗАДАНИЯ (exercise) | = числу упражнений | — |
| Вопросов в «Самопроверке» лекции | **10** | 10–15 |

Если материала меньше 2000 строк — модуль **не готов**. Углубляйся: больше
разборов «под капотом», диаграмм (ASCII), типичных ошибок, реальных кейсов,
сравнений архитектур, ссылок на исходники ядра/glibc. Не вода — техническая
плотность.

---

## 1. Файлы модуля

```
content/
  modules/NN-XX-slug.md          # лекция (NN — порядковый номер, XX — id модуля)
  quizzes/XX.json                # квиз
  exercises/XX/<NN-slug>/        # по папке на упражнение
      problem.md                 # условие
      starter.(c|cpp)            # заготовка с TODO
      test.(c|cpp)               # тесты с main(), exit 0 = успех
  manifest.json                  # регистрация модуля (см. §6)
```

`XX` — id модуля (`f1`, `c1`, `k2`, `el3`, …). Имя файла лекции **обязано**
содержать id в нижнем регистре (`05-C1-...` → стем содержит `c1`): прогонщик
ищет текст модуля для ИИ-наставника по подстроке id в имени файла.

## 2. Лекция (`modules/NN-XX-slug.md`)

### 2.1 ЗАПРЕЩЕНО: YAML frontmatter

Лекция рендерится напрямую через `marked.parse()` (см. `web/app.js`),
**frontmatter не вырезается**. Блок `---\n...\n---` в начале файла превратится
в горизонтальную черту + сырой текст. **Начинай файл сразу с `# Заголовок`.**
Метаданные (что читать, время) — обычным текстом в теле.

### 2.2 Скелет лекции

```markdown
# Модуль XX — Название

> Этап N, <название этапа>. Метка трека: *(...)*. 1–2 абзаца: зачем модуль,
> что студент будет уметь, для кого «проматывай к концу».
>
> [для этапа 2A] **Язык — двуязычный**: основной C++, рядом C-эквивалент.

**Читать к модулю:**
- <книги/стандарты/man из соответствующего раздела трека>

---

## 0. Карта модуля
<таблица разделов: что | зачем системщику>
**Время на модуль:** N–M часов.
**Что значит «освоено»** (цитата критерия из трека).

---

## 1. <Первый крупный раздел>
### 1.1 ...
### 1.2 ...
## 2. ...
...
## N. <последний технический раздел>

---

## <N+1>. Практика и самопроверка
### <N+1>.1 Практические задания   # перечисли упражнения с краткой целью
### <N+1>.2 Вопросы для самопроверки # >= 10 вопросов «ответь вслух»

## <N+2>. Банк вопросов            # карта тем под уровни квиза (БАЗА/МЕХ/ЭКСПЕРТ)

## <N+3>. Что дальше               # связь со следующими модулями
```

Ориентир по структуре и плотности — `modules/03-F3-...md` (1676 строк) и
`modules/05-C1-...md`.

### 2.3 Markdown-ограничения (иначе «поедет»)

- Никакого frontmatter (см. 2.1).
- Кодовые ограждения ` ``` ` — строго парны (проверка: число ` ``` ` чётно).
- Указывай язык у блоков: ` ```cpp `, ` ```c `, ` ```sh `, ` ```text `.
- Рендер — GFM (`marked` с `gfm:true, breaks:false`): таблицы GFM работают,
  переносы строк — только через пустую строку.

### 2.4 Язык кода (политика)

| Этап | Язык кода в лекции и упражнениях |
|------|---------------------------------|
| Этап 1 (F1–F4) | **C** (C17) |
| Этап 2A (C1–C6, userspace) | **двуязычно**: основной **C++** (`std::atomic`, `std::thread`, RAII), рядом **C-эквивалент** (`<stdatomic.h>`, pthread) с пометкой «реальность ядра» |
| Этап 2B (ядро, K1–K7) | **C** (модули ядра) |
| Этап 2C (embedded, EL*) | **C** |

Для 2A: каждый ключевой листинг даёт C++ как основной и C-эквивалент рядом.
Подчёркивай: модель памяти C11 и C++11 одна, меняется синтаксис.

## 3. Квиз (`quizzes/XX.json`) — ТОЧНАЯ СХЕМА

Рендерит `web/quiz.js`. Любое отклонение от схемы = квиз не грузится.

```json
{
  "module": "xx",
  "title":  "XX — Название",
  "levels": {
    "БАЗА":      { "desc": "...", "questions": [ <mcq>, ... ] },
    "МЕХАНИЗМЫ": { "desc": "...", "questions": [ <self_grade>, ... ] },
    "ЭКСПЕРТ":   { "desc": "...", "questions": [ <self_grade>, ... ] },
    "ЗАДАНИЯ":   { "desc": "...", "questions": [ <exercise>, ... ] }
  }
}
```

- `levels` — **объект** (не массив), ключи — ровно эти 4 строки и в этом порядке.
- У каждого уровня — `desc` (строка) и `questions` (массив).
- У вопросов **нет** поля `id` — не добавляй (quiz.js синтезирует id сам).

**Тип `mcq`** (используется в БАЗА):
```json
{ "type": "mcq",
  "text": "Вопрос с `кодом` и **жирным**",
  "options": ["вариант 0", "вариант 1", "вариант 2", "вариант 3"],
  "answer": 2,                          // ИНДЕКС правильного варианта (0-based)
  "explanation": "Развёрнутый разбор, почему именно этот." }
```
`options` — массив **строк**. `answer` — целочисленный индекс в `options`.

**Тип `self_grade`** (МЕХАНИЗМЫ, ЭКСПЕРТ):
```json
{ "type": "self_grade",
  "text": "Открытый вопрос: объясни/докажи/сравни...",
  "hint": "Короткая подсказка — ключевые слова ответа.",
  "reference": "Подробный эталонный ответ (3–8 предложений), по которому студент сам себя оценивает." }
```

**Тип `exercise`** (ЗАДАНИЯ — по одному на каждое упражнение модуля):
```json
{ "type": "exercise", "exercise_id": "01-slug", "title": "Название как в manifest" }
```

Форматтер текста понимает только `` `код` ``, `**жирный**`, перевод строки.
Markdown-таблицы/заголовки внутри строк JSON НЕ рендерятся.

## 4. Упражнения (`exercises/XX/<NN-slug>/`)

### 4.1 Как прогонщик собирает (см. `server.py`)

Пользовательский код пишется во временный каталог как `solution.c`/`solution.cpp`
и компилируется **вместе** с `test.c`/`test.cpp` упражнения:

- Есть `test.cpp` → `g++ -std=c++20 -Wall -Wextra -fsanitize=address,undefined -O1 -g -pthread`
- Есть `test.c`  → `gcc -std=c17 -Wall -Wextra -Wconversion -Wsign-conversion -fsanitize=address,undefined -O1 -g`

Успех теста = программа завершилась с кодом **0**. Таймаут 10 c, CPU-лимит 10 c.

### 4.2 ЖЁСТКИЕ правила

1. **Никаких отдельных заголовков (`.h`), которые включает `solution`.**
   `solution.*` компилируется из временного каталога и `#include "foo.h"` из
   каталога упражнения **не найдёт**. (Из-за этого был сломан `f3/02`.)
   Общие типы определяй прямо в `starter` и `test`, либо используй
   **opaque-указатель**: класс/структура раскрыты только в `solution`, а `test`
   видит лишь forward-declaration и функции-обёртки с `extern "C"`.
2. `test.*` содержит `main()`, печатает `[OK]/[FAIL]` по пунктам и
   `return (pass==run)?0:1;`. Шаблон макроса `CHECK` — как в существующих тестах.
3. `starter.*` компилируется, но тесты на нём **проваливаются** (заглушки
   `return -1; /* TODO */`). Эталонное (правильное) решение — **проходит**.
4. Тесты должны быть **детерминированы** под ASan/UBSan. Гонки ASan не ловит —
   в `problem.md` всегда добавляй команду ручного прогона TSan.
5. Имя папки: `NN-slug` (`01-spsc-queue`). `NN` определяет порядок.

### 4.3 `problem.md`

Заголовок `# Задание: ...`, затем: проблема/мотивация, интерфейс (сигнатуры),
что реализовать (с псевдокодом/подсказками по ordering), «почему так» (для
доказательных тем), для 2A — блок «C-эквивалент», и раздел «Проверка» с
командой TSan/perf. Объём — содержательный, не одна строка.

### 4.4 Тип LKM (модули ядра, этап 2B)

Для упражнений ядра в manifest у упражнения ставится `"type": "lkm"`, рядом с
кодом — `qemu_test.sh`; запуск идёт через QEMU (`/api/lkm/run`), а не gcc.

## 6. Регистрация в `manifest.json`

Найди модуль в нужном `stage` и заполни:

```json
{
  "id": "xx",
  "title": "XX. Название",
  "tag": "новое",                              // или "закрепить" / "частично есть"
  "file": "modules/NN-XX-slug.md",
  "quiz": "quizzes/xx.json",
  "lang": "cpp",                               // для этапа 2A; иначе опусти
  "exercises": [
    {"id": "01-slug", "title": "Название"},
    {"id": "02-slug", "title": "Название"}
    // для LKM: добавь "type": "lkm"
  ],
  "status": "ready"                            // было "planned"
}
```

`title` упражнения в manifest, в `problem.md` и в квизе (`ЗАДАНИЯ`) — согласованы.

## 7. Частые ошибки (реально случались)

- **YAML frontmatter в лекции** → «md поехала». Убери.
- **Квиз со схемой `sections`/`q.correct`/`options:[{id,text}]`** → не грузится.
  Правильно — `levels` / `answer` (индекс) / `options:[строки]` (см. §3).
- **`#include "x.h"` в solution** → ошибка компиляции на сервере. Opaque-указатель.
- **Лекция < 2000 строк** → модуль не готов, углубляйся.
- **Тест зависит от гонок/таймингов** → флапает под ASan. Делай детерминированным.
