# Задание: счётчик вызовов функции через kprobe

Первый шаг в трассировку из ядра: **динамически** прицепиться к функции ядра без её
перекомпиляции и посчитать вызовы. **kprobe** ставит обработчик на вход функции
(через breakpoint/`int3`, §6.1), и твой `pre_handler` срабатывает при каждом вызове.

Здесь ты считаешь вызовы `do_sys_openat2` (путь `openat(2)` — дёргается любым
открытием файла). Символ — **параметр модуля**, так что тем же модулем можно
нацелиться на функцию своего драйвера: `insmod cppmod.ko symbol=vfs_read`.

## Что реализовать

Регистрация kprobe, per-CPU счётчик и `/proc/k7_kprobe` уже даны. Допиши **только**
тело `pre_handler` (в этом и весь урок — запись на hot path без локов):

```c
this_cpu_inc(hits);     // инкремент СВОЕГО per-CPU счётчика
```

## Ключевые API

- **`register_kprobe(&kp)` / `unregister_kprobe(&kp)`** — поставить/снять probe;
  `kp.symbol_name` резолвится в адрес через kallsyms.
- **`pre_handler(struct kprobe *p, struct pt_regs *regs)`** — вызывается ПЕРЕД
  оригинальной инструкцией; возврат `0` = продолжить нормально.
- **`this_cpu_inc` / `per_cpu` / `for_each_possible_cpu`** — per-CPU счётчики (K2 §9).

## Почему per-CPU, а почему unregister обязателен

- **per-CPU, без локов.** `pre_handler` исполняется в **атомарном контексте** той
  функции, что трассируешь (§1.3, §7.4) — спать и брать мьютексы нельзя, а функции
  ядра зовутся параллельно на многих CPU. per-CPU счётчик инкрементируется без
  контеншна; агрегируешь сумму **только** на редком чтении `/proc` (как K6 §15.2).
- **`unregister_kprobe` в `exit` ОБЯЗАТЕЛЕН.** Probe — это breakpoint по адресу в
  ядре, отдающий управление в **твой** `pre_handler`. Не снимешь перед выгрузкой кода
  модуля → breakpoint прыгнет в освобождённую память → oops. Порядок: сначала
  `unregister_kprobe`, потом `proc_remove`.

## Хук в atomic context — спать нельзя

В `pre_handler`: никаких `mutex`/`GFP_KERNEL`/`msleep`/`copy_to_user`. Только
per-CPU/атомики. Тест грепает `dmesg` на `scheduling while atomic` — это поймает
нарушение.

## Проверка

QEMU: модуль грузится, тест читает счётчик из `/proc/k7_kprobe`, делает несколько
`cat` (открытия файлов → вызовы `do_sys_openat2`), снова читает — счётчик должен
**вырасти**. После — `rmmod`, проверка, что `/proc` удалён, и грep `dmesg`.

Самостоятельно: нацель на свою функцию (`symbol=...`) и сравни с тем, что покажет
`funccount 'do_sys_openat2'` (bcc) или `bpftrace -e 'kprobe:do_sys_openat2 {@=count();}'`.
