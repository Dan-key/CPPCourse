# Задание: фильтр по аргументу функции в kprobe

Считать вызовы (упражнение 01) — грубо. Настоящая трассировка смотрит **аргументы**:
«сколько `openat` именно от текущего каталога», «какие байты пишет драйвер». Аргументы
функции лежат в регистрах по ABI (Ф2 §1.1); переносимо их достаёт
`regs_get_kernel_argument` (§7.2).

Здесь ты вешаешь kprobe на `do_sys_openat2(int dfd, ...)`, читаешь 1-й аргумент
(`dfd`) и считаешь **только** открытия относительно текущего каталога
(`dfd == AT_FDCWD`) — это все обычные `open()/cat` без `*at()`-семантики.

## Что реализовать

Регистрация kprobe, per-CPU счётчик и `/proc/k7_arg` уже даны. Допиши тело
`pre_handler`:

```c
int dfd = (int)regs_get_kernel_argument(regs, 0);   // 1-й аргумент: dfd
if (dfd == AT_FDCWD)                                 // открытие относительно cwd
	this_cpu_inc(cwd_opens);
```

## Ключевые API

- **`regs_get_kernel_argument(regs, n)`** — n-й аргумент функции (абстрагирует
  архитектурный ABI: на x86-64 1-й — `rdi`, и т.д.). Безопаснее, чем руками `regs->di`.
- **`AT_FDCWD`** (= -100, `<linux/fcntl.h>`) — «относительно текущего рабочего
  каталога»; `openat(AT_FDCWD, path, ...)` — это обычный `open(path, ...)`.
- per-CPU счётчик + `/proc` — как в упражнении 01.

## Почему через хелпер, а не `regs->di`

`regs->di` — x86-специфика (SysV ABI). `regs_get_kernel_argument` работает и на
arm64, и читается как «n-й аргумент», а не «такой-то регистр». Для переносимого
трассировочного кода — всегда хелпер; ручной доступ к регистрам — только когда точно
знаешь архитектуру и зачем.

## Контекст и владение (повтор, критично)

`pre_handler` — атомарный контекст (§7.4): только per-CPU/атомики, без сна. `kprobe`
снимается в `exit` (`unregister_kprobe`) **до** `proc_remove` — иначе breakpoint в
освобождённой памяти.

## Проверка

QEMU: модуль грузится, тест читает счётчик, делает несколько `cat /proc/version`
(открытия по абсолютному пути идут с `dfd == AT_FDCWD`), снова читает — счётчик
`cwd_opens` должен **вырасти**. После — `rmmod` и грep `dmesg`.

Самостоятельно: сравни с `bpftrace -e 'kprobe:do_sys_openat2 /arg0 == -100/ {
@=count(); }'` (тот же фильтр по аргументу, но в eBPF — и безопаснее: верификатор не
даст разыменовать кривой указатель, §9).
