# Задание: модификация TTL с пересчётом чексуммы

Фильтрация — это чтение. **Mangling** — это **запись** в пакет, и тут две ловушки,
которых нет при чтении: **writable skb** (нельзя писать в разделяемый буфер) и
**чексуммы** (изменил поле — обязан пересчитать). Здесь ты делаешь то, что делает
каждый маршрутизатор на каждом хопе: уменьшаешь **TTL** исходящих пакетов на 1.

Хук стоит на `LOCAL_OUT` (локально сгенерированные пакеты). После загрузки модуля
`ping` к любому адресу покажет TTL ответа на 1 меньше.

## Что реализовать

В `ttl_hook` уже отсеяны skb без IP и «умирающие» (`ttl <= 1`). Допиши три шага —
**порядок и переполучение указателя критичны**:

```c
// 1. сделать заголовок writable (расшарит буфер, если он cloned/shared):
if (skb_ensure_writable(skb, sizeof(struct iphdr)))
	return NF_DROP;                 // нет памяти — дроп
// 2. ПЕРЕПОЛУЧИТЬ iph: буфер мог быть перевыделен, старый указатель невалиден!
iph = ip_hdr(skb);
// 3. уменьшить TTL и инкрементально пересчитать IP-чексумму:
ip_decrease_ttl(iph);              // ttl-- ; iph->check обновлён
```

## Почему именно так

**Writable перед записью (§8.1, §9).** skb можно дёшево **клонировать**: `tcpdump`/
`AF_PACKET` берут клон, **разделяя** буфер данных. Если буфер `skb_cloned`/
`skb_shared`, запись в него **испортит чужую копию** — невоспроизводимые баги,
уязвимости. `skb_ensure_writable(skb, len)` при необходимости делает приватную копию
первых `len` байт (COW сетевого стека). Читать (фильтровать) можно всегда; писать —
только после этого.

**Переполучение `iph` (§8.1, баг #5 из §17.1).** `skb_ensure_writable` мог
**перевыделить** буфер — старый `iph` теперь висит в освобождённой памяти
(use-after-free). Всегда заново `iph = ip_hdr(skb)` после него.

**Чексумма (§8.2–8.3).** IP-заголовок несёт контрольную сумму `iph->check`. Изменил
TTL — обязан пересчитать, иначе пакет отбросит получатель/железо. `ip_decrease_ttl()`
делает это **инкрементально** (правит только дельту через `csum_replace2`) — дёшево
и правильно с учётом `ip_summed` (offload, §8.7). Писать `iph->check` руками,
игнорируя offload, — источник «иногда пакеты бьются».

## Хук в softirq — спать нельзя

`skb_ensure_writable` использует `GFP_ATOMIC` под капотом — спать нельзя (§1.4).
Никаких `GFP_KERNEL`/`mutex`/`msleep`.

## Проверка

QEMU: тест запоминает базовый TTL (`ping 127.0.0.1`, обычно `ttl=64`), грузит
модуль и проверяет, что ответ приходит с `ttl=63` — то есть TTL уменьшен **и**
пакет принят (значит чексумма корректна; с битой чексуммой стек бы его отбросил).
После `rmmod` TTL возвращается к базовому. Финальный грep `dmesg` на
`BUG:/Oops/WARNING:`.

Если бы ты менял адрес/порт (NAT-подобно), добавил бы `csum_replace4` для IP и
`inet_proto_csum_replace4` для L4-чексуммы (§8.4) — здесь достаточно TTL.
