# Задание: блокировка TCP по порту назначения

Дроп целого протокола (упражнение 01) — грубо. Настоящий фильтр смотрит **глубже**:
до транспортного заголовка. Здесь ты блокируешь TCP-соединения на **конкретный
порт**, заданный **параметром модуля** `block_port` (его можно менять на лету через
`/sys/module/cppmod/parameters/block_port`).

Главная сложность — **безопасно** прочитать TCP-заголовок: пакет приходит из сети,
это недоверенный ввод, он может быть короче, чем ты думаешь, или нелинейным
(§2.4, §7.3). Слепое разыменование `tcp_hdr(skb)` — выход за границы (KASAN/oops,
дыра в безопасности).

## Интерфейс

```c
static int block_port;                 // 0 = выключено
module_param(block_port, int, 0644);   // /sys/module/cppmod/parameters/block_port
```

## Что реализовать

В `hook_fn` уже отсеяны не-TCP пакеты. Допиши:

1. **Безопасно** достать TCP-заголовок (работает и для нелинейных пакетов):
   ```c
   struct tcphdr _tcph, *tcph;
   tcph = skb_header_pointer(skb, iph->ihl * 4, sizeof(_tcph), &_tcph);
   if (!tcph)
       return NF_ACCEPT;        // пакет короче — не наш случай
   ```
2. Сравнить порт назначения с `block_port` и при совпадении дропнуть:
   ```c
   if (ntohs(tcph->dest) == block_port) {
       if (net_ratelimit())
           pr_info("k6: drop TCP ->:%d\n", block_port);
       return NF_DROP;
   }
   ```

## Почему `skb_header_pointer`, а не `tcp_hdr(skb)`

- На `LOCAL_IN`/`PRE_ROUTING` транспортные `*_hdr(skb)` могут быть ещё не выставлены
  (заголовки парсятся выше по стеку) — надёжнее считать смещение от IP:
  `iph->ihl * 4` (длина IP-заголовка в байтах).
- `skb_header_pointer` сам проверяет, что байты доступны: вернёт указатель в
  линейную часть, если данные там, иначе **скопирует** их в твой буфер `_tcph`.
  Возврат `NULL` = пакета такой длины нет → не наш случай.

## Порядок байтов

Всё в заголовках — **network byte order** (big-endian). `tcph->dest` — `__be16`,
поэтому сравнивай через `ntohs(tcph->dest) == block_port` (порт в host order) — это
классический баг «фильтр молча не срабатывает», если забыть конверсию (§7.5).

## Проверка

QEMU: модуль грузится с `block_port=9999`; тест поднимает двух слушателей (`nc -l`)
на портах 9999 и 8888, затем проверяет, что соединение на **9999 не проходит**
(SYN дропается), а на **8888 проходит**. После — `rmmod` и грep `dmesg` на
`BUG:/Oops/WARNING:`.

Поменять порт без перезагрузки модуля:
`echo 8080 > /sys/module/cppmod/parameters/block_port`.
