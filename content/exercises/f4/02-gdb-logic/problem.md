# Задание: GDB — отладка логических ошибок без краша

Это **отладочное** упражнение. Программа компилируется чисто и **не падает**:
ни краша, ни ошибок ASan. Но она возвращает **неправильные результаты** —
в коде **3 логические ошибки**. Найди их пошаговой отладкой в GDB.

## Рабочий процесс

```sh
gcc -std=c17 -g -O0 starter.c test.c -o prog
gdb ./prog
(gdb) break binary_search
(gdb) run
(gdb) print lo
(gdb) print hi
(gdb) print mid     # сравни с тем, что ОЖИДАЕШЬ
(gdb) next          # шагай по строкам
(gdb) continue
```

Логические баги не «светятся» сами — их видно, только если сравнивать
фактические значения переменных с ожидаемыми на каждом шаге.

## Где искать

### `int binary_search(const int *arr, int n, int val)`
Должен вернуть **индекс** найденного элемента (0-based) или `-1`.
Поставь breakpoint, проверь, какое именно значение возвращается при попадании —
не смещено ли оно.

### `int fib(int n)`
`fib(0)=0, fib(1)=1, fib(2)=1, fib(3)=2, ...`. Проверь базовые случаи:
одно из них возвращает не то значение.

### `int str_find_char(const char *s, size_t len, char c)`
Индекс первого вхождения `c` или `-1`. Проверь, с какого индекса
начинается перебор — не пропускается ли первый символ.

## Критерий готовности

`./prog` завершается с кодом 0: `binary_search`, `fib`, `str_find_char`
дают корректные значения на всех тестах.
