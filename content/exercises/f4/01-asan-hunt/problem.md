# Задание: ASan-охота — найди и почини 3 бага по памяти

Это **отладочное** упражнение. Код в `starter.c` компилируется, но содержит
**3 ошибки работы с памятью**. Твоя задача — найти каждую с помощью
AddressSanitizer и GDB, затем исправить так, чтобы все тесты прошли.

## Рабочий процесс

```sh
gcc -std=c17 -fsanitize=address,undefined -g starter.c test.c -o prog
./prog            # ASan напечатает ТИП ошибки и строку
gdb ./prog        # run → bt → печатай переменные, чтобы понять причину
```

ASan выводит точный класс ошибки (`heap-buffer-overflow`,
`stack-buffer-overflow`, `heap-use-after-free`), адрес и стек вызовов —
читай его внимательно, он почти буквально показывает строку.

## Где искать

### `char *str_upper(const char *s)`
Возвращает копию строки в верхнем регистре. Подумай, сколько байт нужно
выделить, чтобы поместился ещё и завершающий `'\0'`.

### `long fill_squares(int *out, int n)`
Заполняет `out[0..n-1]` квадратами. Внимательно посмотри на границу цикла —
не пишет ли он на один элемент дальше, чем выделено.

### `const int *find_max_ptr(const int *arr, int n)`
Возвращает указатель на максимальный элемент. Проблема в том, что указатель
ведёт на память, которая к моменту возврата уже освобождена
(use-after-free). Подумай, что именно должно «пережить» `free`.

## Критерий готовности

`./prog` завершается с кодом 0, ASan молчит, выводится `N / N тестов прошли`.

> Цель не просто «заглушить» ASan, а исправить логику владения памятью:
> правильный размер аллокации, корректные границы, корректное время жизни.
