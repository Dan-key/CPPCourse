# Задание: debounce — схлопнуть пачку событий в одно действие

Классическая задача из мира демонов и UI: события приходят **пачками** (редактор
сохранил файл несколькими `write`, `git checkout` тронул 100 файлов, кнопку
кликнули 5 раз), а **действие** (перечитать конфиг, перерисовать, пересобрать)
надо выполнить **один раз, когда всё утихло**. Это debounce.

## Что реализовать

```cpp
void event(uint64_t now_ms);                       // зафиксировать событие
bool poll(uint64_t now_ms, uint64_t* fire_time);   // пора ли «выстрелить»? (один раз)
bool next_fire(uint64_t* out) const;               // когда ближайшая сработка (для timerfd)
```

Семантика (тишина = `quiet_ms`, задаётся в конструкторе):
- каждое `event(now)` **сдвигает** дедлайн сработки на `now + quiet_ms` (новое
  событие «перезапускает» отсчёт тишины);
- `poll(now)` возвращает `true` **ровно один раз**, когда `now >= last_event +
  quiet_ms` и есть несработавшее ожидание; `fire_time = last_event + quiet_ms`;
- после сработки `poll` возвращает `false`, пока не придёт новое `event`.

```cpp
void event(uint64_t now) { last_event_ = now; pending_ = true; }

bool poll(uint64_t now, uint64_t* fire_time) {
    if (pending_ && now >= last_event_ + quiet_) {
        *fire_time = last_event_ + quiet_;
        pending_ = false;                  // «выстрелили» — больше не повторяем
        return true;
    }
    return false;
}
```

## Ключевая идея: пачка схлопывается

Если события идут 100, 110, 120 при `quiet=50`, то сработка — **одна**, в момент
170 (`120 + 50`), а не три. Именно это и нужно: не дёргать дорогое действие на
каждое мелкое событие, а сделать его раз, когда поток затих.

## Связь с timerfd (C4)

В реальном loop: на каждое `event` перевзводишь `timerfd_settime` на
`next_fire()`; когда `timerfd` сработал — выполняешь отложенное действие. Так
сотня событий за секунду превращается в один `timerfd` и одно действие. Это
ровно «демон, перечитывающий конфиг по `inotify`, но не на каждый чих» (мост к
упражнению `06-inotify-dirwatch` + этому).

## Проверка

Автопрогон (ASan/UBSan): одиночное событие срабатывает после тишины ровно раз;
пачка `100,110,120` схлопывается в одну сработку в `170`; после сработки тишина
не даёт ложных срабатываний; новое событие начинает новый цикл.
