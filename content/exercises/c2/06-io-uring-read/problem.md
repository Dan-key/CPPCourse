# Задание: сырой io_uring — одно чтение через кольца

io_uring — это **completion-модель**: ты кладёшь запрос «прочитай» в кольцо
отправки (SQ), ядро выполняет I/O асинхронно и кладёт результат в кольцо
завершений (CQ). Здесь ты сделаешь это **без liburing** (её нет в окружении) —
на голых сисколлах, чтобы увидеть механику до конца.

> **Связь с C1:** кольца SQ/CQ — это SPSC-очереди из §11.1 модуля C1, разделяемые
> между тобой и ядром через `mmap`. Индексы синхронизируются ровно теми
> `acquire`/`release`, что ты там доказывал. Ты публикуешь SQE (release на
> `sq_tail`), ядро читает (acquire); ядро публикует CQE (release на `cq_tail`),
> ты читаешь (acquire).

## Что реализовать

Вся настройка колец (`io_uring_setup` + три `mmap`) **уже сделана** в `Ring::init`.
Тебе — заполнить функцию `uring_read`: выполнить **одно** чтение `fd` в `buf` и
вернуть результат.

```cpp
int uring_read(Ring& r, int fd, void* buf, unsigned len);
```

Пять шагов (подробно расписаны в комментарии стартера):
1. взять свободный слот SQE по `sq_tail & sq_mask`;
2. заполнить операцию: `opcode = IORING_OP_READ`, `fd`, `addr = buf`, `len`,
   `off = 0`, `user_data`, и `sq_array[idx] = idx`;
3. **опубликовать** SQE: `sq_tail.store(tail + 1, release)`;
4. один сисколл `io_uring_enter(ring_fd, 1, 1, IORING_ENTER_GETEVENTS)` —
   «отправь 1, дождись 1 завершения»;
5. забрать CQE: прочитать `cqe.res` (это число байт, как у `read`: `>0` данные,
   `0` EOF, `<0` это `-errno`), затем продвинуть `cq_head.store(head+1, release)`.

`cqe.res` — это **готовый результат**: ядро **уже** прочитало данные в `buf`. Это и
есть completion-модель (в отличие от epoll, который лишь говорит «готов, читай
сам»).

## Почему это важно

Раз увидев голые кольца, ты понимаешь io_uring насквозь: откуда берётся экономия
сисколлов (можно положить много SQE и сделать один `io_uring_enter`; с SQPOLL —
вообще без сисколла), и почему он умеет async для файлов, чего epoll не может.

В реальном коде вместо этой механики бери **liburing** (`io_uring_prep_read`,
`io_uring_submit`, `io_uring_wait_cqe`) — те же кольца, но по-человечески.

## Проверка

Автопрогон: в pipe записаны данные, `uring_read_once` должен их прочитать через
io_uring и вернуть число байт; затем — EOF после закрытия. Если ядро **не
поддерживает** io_uring (старое/запрещён), тест печатает `[SKIP]` и засчитывается
(не валим из-за окружения).

Вживую посмотреть io_uring обычным `strace` почти нельзя (вся работа в кольцах) —
используй `sudo perf trace -e 'io_uring:*' ./prog`.
