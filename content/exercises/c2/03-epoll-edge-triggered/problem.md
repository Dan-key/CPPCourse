# Задание: epoll эхо-сервер (edge-triggered) — грабли ET

То же, что упражнение 02, но в **edge-triggered** режиме (`EPOLLET`). Здесь ты на
практике почувствуешь **главную ошибку ET**, которую спрашивают на собеседованиях.

## Чем ET отличается

LT сообщает о готовности, **пока** в сокете есть данные. ET сообщает **только на
фронте** — когда пришли **новые** данные. Получив одно `EPOLLIN`, ты обязан
**прочитать всё, что есть, в цикле до `EAGAIN`**. Если прочитаешь только часть —
остаток «зависнет»: второго события не будет, пока не придёт **ещё** порция.

```
Клиент прислал 50 000 Б одним куском → ОДНО событие EPOLLIN.
Ты прочитал 8 192 Б и остановился.
Следующий epoll_wait НИЧЕГО не вернёт — оставшиеся 41 808 Б потеряны.
```

## Что реализовать

```cpp
int echo_server_et(int listen_fd, int stop_fd);
```

Те же обязанности, что в 02, но:
- `listen_fd` и каждое соединение регистрируй с `EPOLLIN | EPOLLET`;
- на событие соединения — **цикл `read` до `EAGAIN`**, каждый кусок сразу пишешь
  обратно;
- `accept` — тоже **в цикле до `EAGAIN`** (иначе непринятые соединения зависнут);
- ET **требует** неблокирующих fd (они уже такие) — иначе последний `read`
  заблокирует весь loop.

```cpp
// Обработка события соединения в ET:
for (;;) {
    char buf[8192];
    ssize_t r = read(fd, buf, sizeof buf);
    if (r > 0)       { /* записать r байт назад */ }
    else if (r == 0) { epoll_ctl(ep, EPOLL_CTL_DEL, fd, nullptr); close(fd); break; } // EOF
    else { if (errno == EAGAIN) break;          // дренаж завершён — ждём след. фронт
           if (errno == EINTR)  continue;
           epoll_ctl(ep, EPOLL_CTL_DEL, fd, nullptr); close(fd); break; }
}
```

## Проверка

Автопрогон (ASan/UBSan): клиенты шлют **большие** сообщения (50 000 Б — заведомо
больше одного `read`-буфера). Корректное ET-эхо требует читать до `EAGAIN`; если
дренажа нет — эхо неполное и тест провалится (или повиснет → таймаут). Прочувствуй
разницу: временно убери цикл (`read` один раз) — увидишь потерю данных.

Вживую: `scripts/io-playground.py source --port <свой> --bytes 10000000` льёт
поток — под `strace -e read` увидишь серию `read`, заканчивающуюся `EAGAIN`.
