# Задание: динамический per-CPU счётчик (alloc_percpu)

Счётчик на горячем пути, который пишут с многих ядер, **не должен** быть общим
`atomic_t` — общая строка кэша пинг-понгует между ядрами (C1 §10) и не масштабируется.
Решение — **per-CPU**: по экземпляру на каждое ядро; инкремент локален (`this_cpu_add`,
без локов и без межъядерного трафика), а сумма считается **редко** на чтении.

В K2 ты использовал **статический** `DEFINE_PER_CPU`. Здесь — **динамический**
аллокатор: **`alloc_percpu`**/`free_percpu` (так делают, когда число per-CPU структур
известно лишь в рантайме). Строишь `/dev/cpppc`: `write "<N>"` прибавляет N к счётчику
**своего** ядра, `read` агрегирует по **всем** ядрам.

## Что реализовать

- **`init`** — `counter = alloc_percpu(long)`; если `NULL` — `-ENOMEM`; затем
  `misc_register`.
- **`write("<N>")`** — распарсить число и `this_cpu_add(*counter, n)` (на своём ядре).
- **`read`** — `for_each_possible_cpu(cpu) total += *per_cpu_ptr(counter, cpu)`; отдать
  `total`.
- **`exit`** — `misc_deregister` → `free_percpu(counter)`.

## Ключевые API

- **`alloc_percpu(type)`** — выделить по экземпляру `type` на каждое **возможное** ядро;
  вернёт `type __percpu *` (или `NULL`).
- **`this_cpu_add(*ptr, v)`** — прибавить к экземпляру **текущего** ядра (preempt-safe,
  без локов).
- **`per_cpu_ptr(ptr, cpu)`** — указатель на экземпляр **конкретного** ядра (для
  агрегации).
- **`for_each_possible_cpu(cpu)`** — обход всех возможных ядер (не `online` — иначе
  потеряешь вклад ушедшего offline ядра).
- **`free_percpu(ptr)`** — освободить.

## Почему per-CPU масштабируется

`this_cpu_add` пишет в память **своего** ядра → ноль когерентного трафика (C1 §4.2),
ноль контеншна. Цена — память (N экземпляров) и стоимость **чтения** (обход ядер,
редкий). Идеально для «пишут часто из многих ядер, читают редко» (статистика). Динамика
(`alloc_percpu`) — когда таких счётчиков много и число их рантайм-зависимо.

## Проверка

QEMU: `echo 5 > /dev/cpppc; cat` → `5`; ещё `echo 3` → `8`; `echo -2` → `6`; сто раз
`echo 1` → `106`. `dmesg` чист.
