# Задание: атомарный счётчик с lock-free отслеживанием максимума

Ядровый аналог lock-free паттерна из C1: счётчик, который растёт атомарно, плюс
«исторический максимум», обновляемый **без блокировки** — циклом
`atomic_cmpxchg` (compare-and-swap ядра). На SMP плоский `int` здесь потерял бы
обновления; `atomic_t` + CAS корректны без спинлока.

## Устройство `/dev/cppmax`

- `write "<N>"` — атомарно прибавить `N` (можно отрицательное) к счётчику и
  обновить максимум.
- `read` — строка `counter=<v> max=<m>\n`.

## Что реализовать

### `bump_high(int v)` — обновить максимум через CAS-цикл
```c
int old = atomic_read(&high);
while (v > old) {
    int prev = atomic_cmpxchg(&high, old, v);  // если high == old → записать v
    if (prev == old) break;                     // успех
    old = prev;                                  // кто-то опередил — перечитали
}
```
Это ровно lock-free «обновить максимум» из C1, но на ядровых атомиках. CAS
возвращает **прежнее** значение; равно `old` → мы выиграли.

### `max_write` — `v = atomic_add_return(N, &counter); bump_high(v);`
### `max_read` — отформатировать `atomic_read(&counter)` и `atomic_read(&high)`.

## Почему именно так (теория K2)

- `atomic_t` — это тип для счётчиков, видимых нескольким ядрам/контекстам;
  `atomic_add_return`, `atomic_cmpxchg` — неделимые RMW (как в C1, но через
  ядровое API).
- **Без спинлока:** обе операции атомарны сами по себе, а максимум собирается
  CAS-циклом — поэтому код корректен на SMP без блокировки и без сна (его можно
  звать даже в атомарном контексте).
- Ядровые атомики несут **барьеры**: `atomic_cmpxchg` — полный барьер;
  «расслабленные» версии — `atomic_cmpxchg_relaxed` (как `memory_order_relaxed`,
  C1 §5).

## Сборка и проверка

Модуль собирается как `cppmod.ko` против заголовков ядра и грузится `insmod` в
QEMU. Тест (`qemu_test.sh`): `echo` чисел в `/dev/cppmax`, проверка строки
`read`, и что `dmesg` не ругается (`BUG`/`WARNING`/`scheduling while atomic`).
Запуск — кнопкой «⚙ Собрать и запустить в QEMU» в редакторе (нужен настроенный
`scripts/qemu-setup.sh`).
