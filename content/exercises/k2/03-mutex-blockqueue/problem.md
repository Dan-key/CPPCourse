# Задание: блокирующая очередь — мьютекс + wait queue

Реализуй очередь сообщений, где **читатель засыпает**, пока нет данных, а
писатель его будит. Это контраст к спинлоку (упр. 02): **мьютекс может спать**, а
**wait queue** — это ядровый аналог condition variable из C2 §17.

## Устройство `/dev/cppq`

- `write "<msg>"` → положить сообщение, разбудить ждущего читателя
- `read` → вернуть самое старое; если пусто — **уснуть** до появления

## Что реализовать

### `q_write` — положить и разбудить
```c
mutex_lock(&qlock);
if (q_count < QCAP) {
    strscpy(q[q_tail], tmp, MSGMAX);
    q_tail = (q_tail + 1) % QCAP;
    q_count++;
}
mutex_unlock(&qlock);
wake_up_interruptible(&qwait);            // разбудить спящий read
```

### `q_read` — уснуть по предикату, затем снять
```c
if (wait_event_interruptible(qwait, READ_ONCE(q_count) > 0))
    return -ERESTARTSYS;                  // прерван сигналом — корректно выйти
mutex_lock(&qlock);
if (q_count == 0) { mutex_unlock(&qlock); return 0; }  // другой read опередил
strscpy(tmp, q[q_head], MSGMAX);
q_head = (q_head + 1) % QCAP;
q_count--;
mutex_unlock(&qlock);
```

## Мьютекс vs спинлок — ключевая разница K2

| | Спинлок (упр. 02) | Мьютекс (это упр.) |
|--|--|--|
| Ожидание | busy-wait (крутится) | спит (планировщик переключает) |
| Можно спать под локом? | **НЕТ** | **ДА** |
| Контекст | любой (в т.ч. прерывание) | только тот, где можно спать (process context) |
| Когда | очень короткая секция | длинная секция / нужно спать |

`wait_event_interruptible(wq, COND)` — засыпает, **пока `COND` ложно**, и
просыпается, когда `wake_up_interruptible(&wq)` дёрнут И `COND` стало истинно.
Это предикат-цикл из C2 §17, встроенный в макрос (защита от ложных пробуждений).
`-ERESTARTSYS` при прерывании сигналом — правильный ответ ядра (сисколл
перезапустится или вернёт `EINTR`).

> Почему `READ_ONCE(q_count)` в условии: `q_count` читается вне мьютекса в
> предикате `wait_event` — `READ_ONCE` запрещает компилятору «соптимизировать»
> повторное чтение (ядровый аналог `volatile`-доступа; см. лекцию про барьеры).

## Сборка и проверка

`cppmod.ko` в QEMU; тест проверяет FIFO-порядок и **блокирующее** чтение
(читатель в фоне спит → `echo` будит его), плюс чистый `dmesg`.
