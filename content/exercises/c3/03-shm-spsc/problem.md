# Задание: lock-free SPSC очередь в разделяемой памяти (межпроцессно)

Синтез всего этапа: **SPSC-кольцо из C1 §11.1**, но размещённое в **разделяемой
памяти** и работающее **между процессами**. Один процесс-производитель и один
процесс-потребитель обмениваются данными **без единого сисколла** на горячем пути
— только атомарные индексы. Это самый быстрый IPC из возможных, и ровно так
устроены кольца io_uring (C2) между тобой и ядром.

## Что реализовать

Структура `SpscRing` уже задана (в разделяемой памяти; только индексы и массив,
без указателей внутри). Реализуй `spsc_push`/`spsc_pop` — **тот же** код и
ordering, что в C1 `01-spsc-queue`:

```cpp
int spsc_push(SpscRing* q, int v) {
    auto t = q->tail.load(std::memory_order_relaxed);   // свой → relaxed
    auto h = q->head.load(std::memory_order_acquire);   // чужой → acquire
    if (t - h == SpscRing::CAP) return 0;               // полна
    q->buf[t & (SpscRing::CAP - 1)] = v;               // (A) данные
    q->tail.store(t + 1, std::memory_order_release);    // (B) публикуем
    return 1;
}
```
`spsc_pop` симметричен: `head` — relaxed, `tail` — acquire, данные читаем до
сдвига `head`, `head` публикуем release.

## Почему это работает МЕЖДУ ПРОЦЕССАМИ

- Память выделена через `mmap(MAP_SHARED | MAP_ANONYMOUS)` **до** `fork` —
  потомок наследует то же физическое отображение, поэтому оба процесса видят одни
  байты (у них разные виртуальные адреса, но один объект — поэтому внутри
  структуры **нельзя** хранить указатели, только индексы).
- `std::atomic<size_t>` на x86-64 — **lock-free** и работает прямо по байтам
  разделяемой памяти, поэтому синхронизация acquire/release действует через
  границу процесса так же, как между потоками (модель памяти C1 одна и та же).

## C-эквивалент

`_Atomic size_t` + `atomic_*_explicit` в C; та же `mmap(MAP_SHARED)`. Так пишут
межпроцессные кольца в высокопроизводительных системах (торговля, телеметрия),
где даже один сисколл на сообщение — слишком дорого.

## Проверка

Автопрогон (ASan/UBSan): кольцо в анонимной разделяемой памяти; `fork`; **потомок**
— производитель шлёт `0..N-1`, **родитель** — потребитель принимает строго по
порядку через границу процесса. Проверяется FIFO без потерь и штатное завершение
потомка. (Гонки лови локально TSan, как в C1.)
