# Задание: мьютекс на futex (машина состояний 0/1/2)

`futex` («fast userspace mutex») — примитив, на котором в Linux построены
`std::mutex`, `pthread_mutex`, семафоры и межпроцессная синхронизация в
разделяемой памяти. Идея: **без контеншна** захват/освобождение — это атомарный
CAS в userspace **без сисколла**; и только **при контеншне** зовётся `futex(2)`,
чтобы усыпить/разбудить поток в ядре. Это та самая машина состояний из C1 §8.2 —
теперь реализуешь сам.

## Состояния слова мьютекса

```
0 = свободен
1 = занят, ждущих нет
2 = занят, есть ждущие (при unlock надо будить через ядро)
```

## Что реализовать

```cpp
void futex_lock(std::atomic<int>* m);
void futex_unlock(std::atomic<int>* m);
```

### `futex_lock` (алгоритм Дреппера)
```cpp
int c = 0;
if (m->compare_exchange_strong(c, 1, std::memory_order_acquire))
    return;                                   // было 0 → заняли, БЕЗ сисколла (fast path)
if (c != 2)
    c = m->exchange(2, std::memory_order_acquire);   // пометить «есть ждущие»
while (c != 0) {                              // пока занят
    futex(m, FUTEX_WAIT_PRIVATE, 2);          // спим в ядре, пока значение == 2
    c = m->exchange(2, std::memory_order_acquire);
}
```
(`compare_exchange_strong(c, 1)` при неудаче кладёт в `c` текущее значение.)

### `futex_unlock`
```cpp
if (m->fetch_sub(1, std::memory_order_release) != 1) {  // было 2 → есть ждущие
    m->store(0, std::memory_order_release);
    futex(m, FUTEX_WAKE_PRIVATE, 1);          // разбудить одного
}
```
Если было `1` (ждущих нет), `fetch_sub` вернёт 1 → просто освободили, **без
сисколла**.

## Почему это важно

Вся прелесть — в **fast path**: при отсутствии конкуренции это пара атомиков, как
спинлок, но без жжения CPU (при контеншне поток спит). Поэтому «по умолчанию
мьютекс» в userspace почти бесплатен (C1 §8.3). `FUTEX_*_PRIVATE` — оптимизация
для одного процесса; для futex в разделяемой памяти между процессами используют
варианты без `_PRIVATE`.

## C-эквивалент / связь

В C — `_Atomic int` + `syscall(SYS_futex, ...)`. Тот же futex в разделяемой памяти
(`mmap`) синхронизирует РАЗНЫЕ процессы — мост к упражнению 03. acquire/release —
из C1.

## Проверка

Автопрогон (ASan/UBSan): 8 потоков по 100 000 инкрементов общей переменной под
`futex_lock`. Итог обязан быть ровно `THREADS*ITERS`, внутри секции — всегда один
поток, а после всех `unlock` слово мьютекса == 0. Гонки — TSan локально.
