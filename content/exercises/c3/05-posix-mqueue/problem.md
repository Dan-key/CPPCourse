# Задание: POSIX очередь сообщений (`mq_*`)

В отличие от pipe/TCP (поток **байтов**), POSIX message queue хранит сообщения **с
границами** и **с приоритетами**: каждое `mq_send` — отдельное сообщение, а
`mq_receive` всегда отдаёт сообщение с **наибольшим** приоритетом. Удобно для
команд/задач между процессами, где важны границы и важность.

## Что реализовать

```cpp
mqd_t   mq_setup(const char* name);                                  // открыть/создать
int     mq_send_msg(mqd_t q, const char* s, unsigned prio);          // отправить
ssize_t mq_recv_msg(mqd_t q, char* buf, size_t buflen, unsigned* p); // принять
```

### `mq_setup`
```cpp
struct mq_attr attr{};
attr.mq_maxmsg  = 10;     // макс. сообщений в очереди
attr.mq_msgsize = 256;    // макс. размер одного сообщения
return mq_open(name, O_CREAT | O_RDWR, 0600, &attr);   // имя вида "/myqueue"
```

### `mq_send_msg` / `mq_recv_msg`
```cpp
mq_send(q, s, strlen(s), prio);                  // priority — старшие вперёд
mq_receive(q, buf, buflen, prio);                // buflen ДОЛЖЕН быть >= mq_msgsize!
```
Важно: буфер `mq_receive` обязан быть **не меньше** `mq_msgsize` очереди (иначе
`EMSGSIZE`) — поэтому тест передаёт буфер 256 байт.

## Семантика, которую надо понимать

- **Границы сохраняются:** одно `mq_send` ↔ одно `mq_receive` (не «слипаются»,
  как байты в TCP — там нужен фрейминг, C2 §6.7).
- **Приоритеты:** при чтении первым выходит сообщение с наибольшим `prio` (при
  равных — FIFO).
- **Именованная и персистентная:** очередь живёт в ядре (видна в `/dev/mqueue`)
  до `mq_unlink`, как shm-объект.
- **Async-уведомление:** `mq_notify` может слать сигнал/спавнить поток при
  приходе сообщения (в этом задании не нужно).

## C-эквивалент

Идентичные сисколлы в C. Альтернатива — SysV message queues (`msgget`/`msgsnd`/
`msgrcv`, старый API с числовыми ключами и `mtype`). POSIX-вариант новее и удобнее.

## Проверка

Автопрогон (ASan/UBSan): шлём сообщения с приоритетами 1, 5, 3 — принять обязаны в
порядке 5 → 3 → 1 (по приоритету), с правильными телами. Если окружение **не
поддерживает** mqueue, тест печатает `[SKIP]` и засчитывается. Имя очереди
уникально на процесс; в конце — `mq_close` + `mq_unlink`.
