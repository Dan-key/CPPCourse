# Задание: fork/exec/wait — создание потомков и сбор статусов

Научись порождать процессы через `fork(2)`, запускать программы через `exec`, и корректно собирать статусы завершения через `waitpid(2)`.

## Что нужно реализовать

### `int fork_and_collect(int n, int *results)`

Создать `n` дочерних процессов. Потомок с порядковым номером `i` (0-based)
завершается с кодом выхода `i + 1`. Дождаться всех потомков через
`waitpid(-1, ..., 0)` и записать их коды выхода в `results[0..n-1]`
**в порядке завершения** (не обязательно 1,2,3 — порядок диктует планировщик).

- Возвращает `0` при успехе, `-1` при ошибке `fork` или `waitpid`.
- Граничный случай: `n == 0` → вернуть `0` немедленно, без `fork()`.

### `int run_echo(const char *arg)`

Запустить `/bin/echo` с одним аргументом `arg` через `execv()` в дочернем процессе.
Дождаться завершения и вернуть код выхода (`0` при успехе echo), либо `-1` при
ошибке `fork`/`exec`.

### `int check_signal_death(int sig)`

Создать потомка, который вызывает `raise(sig)` (шлёт себе сигнал). Дождаться
через `waitpid()`. Если потомок убит сигналом — вернуть `WTERMSIG` (номер сигнала),
иначе `0`; при ошибке — `-1`.

## Ключевые идеи

```c
pid_t pid = fork();
if (pid < 0)  return -1;        // ошибка
if (pid == 0) { /* потомок */ _exit(code); }
/* родитель */
int status;
waitpid(pid, &status, 0);
if (WIFEXITED(status))   code = WEXITSTATUS(status);
if (WIFSIGNALED(status)) sig  = WTERMSIG(status);
```

- В потомке используй `_exit()`, а не `exit()` — чтобы не запускать atexit-хендлеры
  и не сбрасывать буферы дважды.
- `WEXITSTATUS` хранит только младшие 8 бит кода выхода.

> Совет: запускай с `ASAN_OPTIONS=detect_leaks=0`, чтобы LeakSanitizer не ругался
> на «утечки» в форкнутых потомках.

## Тесты

- `fork_and_collect(3)` → results содержит `{1,2,3}` в любом порядке, все уникальны
- `fork_and_collect(0)` → `0`, без единого `fork()`
- `run_echo("hello")` → `0`
- `check_signal_death(SIGTERM)` → `15`, `SIGKILL` → `9`
